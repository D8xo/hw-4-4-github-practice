# Project Plan

## Goals
- Learn to use Github features such as branches, issues, commits and pull
- Demonstrate proper organization

## Needs
- Structured repo
- Clear doc
- Organized workflow using issues and branches

## Steps
1. Create Rep
2. Create issues
3. Develop changes in seperate branch
4. Commit changes and clear msg
5. Create pull and merge into main

## Repo Maintenance
- Use issues to track
- Use branches for new work
- Write clear commit msg
- Keep README updated
